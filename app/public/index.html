⚙️ Comandos kubectl para Escalado

Utiliza estos comandos en tu terminal para escalar el deployment. Recuerda que estamos trabajando en el namespace gcplab

📈 Escalar UP (aumentar réplicas a 3):
kubectl scale deployment gcp-lab-app --replicas=3 -n gcplab
📉 Escalar DOWN (reducir a 1 réplica):
kubectl scale deployment gcp-lab-app --replicas=1 -n gcplab
👀 Ver todos los pods:
kubectl get pods -n gcplab
📋 Descripción del deployment:
kubectl describe deployment gcp-lab-app -n gcplab
📊 Ver recursos (todo):
kubectl get all -n gcplab
👁️ Monitorear pods en tiempo real:
kubectl get pods -n gcplab --watch

📚 Pasos de Aprendizaje

1 Crear el namespace: Primero, crea el namespace gcplab para organizar tus recursos.
kubectl create namespace gcplab
2 Desplegar la aplicación: Usa kubectl apply para desplegar todos los recursos YAML.
kubectl apply -f devops/
3 Verificar el despliegue: Confirma que el pod se está ejecutando correctamente.
kubectl get pods -n gcplab
4 Escalar el deployment: Aumenta o disminuye el número de réplicas según sea necesario.
kubectl scale deployment gcp-lab-app --replicas=3 -n gcplab
5 Monitorear en tiempo real: Observa cómo se crean y terminan los pods.
kubectl get pods -n gcplab --watch
